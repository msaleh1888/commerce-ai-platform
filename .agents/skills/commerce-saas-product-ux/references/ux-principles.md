# UX Principles

## Product Feel

The app should feel like a premium B2B AI operations platform, not a landing page or generic analytics dashboard.

Target qualities:

- Calm.
- Dense.
- Trustworthy.
- Precise.
- Fast to scan.
- Operationally useful.
- Visually polished without feeling decorative.

## Information Architecture

Prefer a stable SaaS shell:

- Left sidebar navigation.
- Top bar with tenant, search, status, and user controls.
- Main workspace for the current workflow.
- Optional right-side detail, evidence, or activity panel.

Core navigation:

- Dashboard.
- Imports.
- Products.
- Search.
- Review Queue.
- Evaluation.
- Audit.
- Settings.

## Screen Design Rules

- Put the user's next best action near the page title.
- Make tenant context visible.
- Show status, ownership, and consequences before irreversible actions.
- Use tables for scan-heavy operational data.
- Use side panels or detail pages for evidence.
- Show raw data and normalized data where comparison matters.
- Prefer specific labels over generic "AI insights" language.
- Include empty, loading, error, permission-denied, and partial-success states.

## AI UX Rules

- Show what the system knows, what it inferred, and what remains uncertain.
- Cite product fields, source records, retrieval units, or evaluation manifests.
- Keep model suggestions separate from approved business state.
- Make human approval visible for risky mutations.
- Avoid presenting private reasoning as an audit explanation.

## Visual Direction

Use mostly neutral surfaces with restrained accent colors.

Good references:

- Linear for product polish.
- Stripe dashboard for clarity.
- Vercel for technical elegance.
- Shopify Admin for commerce practicality.
- Retool for dense operational workflows.
- Langfuse for AI observability credibility.

Avoid:

- One-note purple AI branding.
- Huge gradient hero sections inside the app.
- Decorative cards nested inside cards.
- Empty dashboards with vague metrics.
- Placeholder data that feels fake.
