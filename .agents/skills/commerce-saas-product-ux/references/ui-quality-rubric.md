# UI Quality Rubric

Use this rubric when reviewing Figma screens, screenshots, or frontend implementation.

## Screen-Level Checks

- The primary user goal is obvious within 5 seconds.
- The primary action is visible and named clearly.
- The page title matches the user's mental model.
- Tenant context is visible.
- Status and progress are easy to scan.
- Dense data remains readable.
- Evidence is close to the decision it supports.
- Risky actions show consequences before confirmation.
- Empty, loading, error, permission-denied, and success states exist.

## Visual Polish Checks

- Spacing follows a consistent scale.
- Typography hierarchy is clear.
- Tables align cleanly and support scanning.
- Badges use consistent color semantics.
- Buttons use icons where helpful.
- Cards are not nested inside other cards.
- Borders and shadows are restrained.
- The palette is not dominated by one trendy AI color.
- Realistic data replaces generic placeholders.

## Professional SaaS Checks

- The UI feels useful before it feels decorative.
- The layout supports repeated work.
- The workflow feels safe and auditable.
- Technical credibility is visible through metrics, config, provenance, and state.
- The app does not depend on marketing copy to explain itself.

## Portfolio Wow Checks

- A reviewer can tell this is not a chatbot.
- The hardest workflow has the most polished screen.
- The design highlights evidence, evaluation, and approval.
- Screenshots would look credible in a README or case study.
- The product story is visible in the interface itself.

## Red Flags

- A dashboard full of vague KPIs.
- AI suggestions with no evidence.
- Search results with no ranking explanation.
- Review actions with no audit trail.
- Import errors hidden from the user.
- Cross-tenant context unclear.
- Placeholder text that breaks the illusion of a real product.
